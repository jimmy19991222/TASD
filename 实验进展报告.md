## 1. 方法框架（当前主线）

**Self-Teacher Advantage**：在 GRPO 的 PG loss 框架内，用 EMA teacher 的 token 概率（或 teacher 在 student top-k 上的 logp）构造 advantage。

| 维度 | 当前选择 |
| --- | --- |
| Loss | GRPO clipped surrogate（PG）|
| Advantage 构造 | `adv_mode=self_teacher` + `use_vce=True` |
| Teacher | EMA，更新率 `r=0.1` |
| Reward | 环境 outcome reward |
| Norm | group-level mean，**未除 std** |
| Clip | `clip_adv_value=3.0` |
| Entropy coef | `0.001` |

详见 [Self_Teacher_Advantage_Method.md](file:///Users/awesome_jimmy/lazada/SDPO/Self_Teacher_Advantage_Method.md)。

## 3. 当前主假说：隐式 V_CE 退化

`self_teacher_advantage` 用 teacher prob 构造 advantage：

- teacher 高概率 token → A > 0
- teacher 低概率 token → A < 0

但 EMA r=0.1 + rel_L2≈1e-4 + cos_sim=1.0 → **teacher 几乎就是 student**：

> teacher 高 prob ⇔ student 高 prob，teacher 低 prob ⇔ student 低 prob

代入 PG 梯度 `A · ∇log π(a)`：

```
高 prob token (A > 0) → 推得更高
低 prob token (A < 0) → 推得更低
```

这就是 **V_CE 的本质**（点级模式追逐），与 SDPO 的分布级 JSD 截然不同 —— SDPO 之所以不崩，是因为它对整个分布形状有约束。

**放大器**：
- group norm + clip=3 → advantage 全程贴边 → 主假说方向被无差别放大
- `use_vce=True` 显式叠加 V_CE 字面项
- entropy_coeff=0.001 太弱
- 负 advantage 稀缺（teacher 几乎处处 ≥ student），剩下主要是正向 mode-seeking

### Tier 1（**新主叙事首发：Prior-Shift POC**）⭐

**核心 idea**：用 teacher 的 forward Bayesian surprise 作为 token-level credit，把 outcome reward 按 surprise 分配。

**公式**：
```
g_t = KL( P_T(·|x, y_≤t) ‖ P_T(·|x, y_<t) )       # token-level surprise
ĝ_t = g_t / Σ_t g_t                                # group-level normalize
A_t = R · ĝ_t · L                                  # L = response length，保持总量
```

**vs RLSD 的差异**：
- RLSD：`A_t = A · (P_T/P_S)^sign(A)` —— **基于 teacher-student diff**
- Ours：`A_t = R · ĝ_t` —— **基于 teacher 自己的 belief update**，与 student 无关
- 解释：RLSD 是 "模仿 teacher"，我们是 "找 teacher 认为信息含量大的位置"

**工程实现要点**：
- 仅需 +1 次 teacher forward（拿 logits over full vocab）
- 复用现有 `entropy_gate` 基础设施作 mask 框架
- 入口：`run_local_tasd.sh prior_shift_v1`（待加）
- adv_mode 新增 `prior_shift`，与 `self_teacher` 并列

**验证指标**：
- `prior_shift/g_t_mean`、`prior_shift/g_t_top10p_ratio`（surprise 集中度）
- `prior_shift/credit_top10p_token_entropy`（高 credit token 是否落在低先验位置）
- 标准三件套：entropy 不崩 + val 不过峰 + response 不缩

### Tier 2（**新主叙事扩展：Posterior-Shift**）

**核心 idea**：teacher 看到 outcome 后反推每个 token 的重要性（hindsight credit）。

**公式**：
```
h_t = log P_T(y_t | x, y, r=R) − log P_T(y_t | x, y_<t)
```

**实现路径**：
- 把 `<reward=R>` 作为 prefix token 喂给 teacher（或在 system prompt 加入 outcome 描述）
- 需要先在 teacher 训练 / inference pipeline 加 reward-conditioned forward 接口
- 与 Tier 1 比，多 1 次 teacher forward

**实验顺序**：
- C1：Posterior-Shift 单独跑
- C2：Prior + Posterior 加权融合（α·g_t + (1−α)·h_t）
- C3：与 RLSD baseline 对比 val_acc / entropy / response_length

## 7. Related Work：RLSD 撞车与 reframe

- **论文**：Self-Distilled RLVR (RLSD)，arXiv:2604.03128v2
- **核心公式**：`Â_t = A · clip((P_T/P_S)^sign(A), 1−ε_w, 1+ε_w)`
- **理论贡献**：Bayesian evidence ratio (Theorem 4)、impossibility trilemma (Theorem 3)、leakage-free 证明 (Theorem 5)、OPSD mutual information gap (Theorem 1)
- **实验**：Qwen3-VL-8B-Instruct + MMFineReason-123K，5 个多模态 benchmark，比 GRPO +2.32%
- **撞车结论**：早期"prob diff + sign(R)"思路与 RLSD 同构（线性近似）；前向 prob ratio 这条路已被堵死

**Reframe 后的差异化定位**：

| 维度 | RLSD | Ours (Prior+Posterior Shift) |
| --- | --- | --- |
| credit 来源 | teacher-student log-ratio | teacher 自己的 belief shift |
| 因果方向 | 前向（causal） | 前向 + 反向（hindsight） |
| 是否需要 student 信号 | 是 | 否（解耦 teacher）|
| Bayesian 视角 | prior 模仿 | prior + posterior 双视角 |
| benchmark | 多模态 Qwen3-VL | 文本任务（sciknoweval / math）|
| teacher 形式 | hard sync 每 10 步 | EMA + 实测对比 |
