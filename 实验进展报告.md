## 1. 方法框架（当前主线）

**Self-Teacher Advantage**：在 GRPO 的 PG loss 框架内，用 EMA teacher 的 token 概率（或 teacher 在 student top-k 上的 logp）构造 advantage。

| 维度 | 当前选择 |
| --- | --- |
| Loss | GRPO clipped surrogate（PG）|
| Advantage 构造 | `adv_mode=prior_shift`（v2: max_ratio=3 + length penalty） |
| Teacher | EMA，更新率 `r=0.05` |
| Reward | 环境 outcome reward |
| Norm | group-level mean，**未除 std** |
| Prior-Shift 公式 | `A_t = A_seq · ĝ_t · length_scale`（见下方） |

### Prior-Shift v2 完整公式

```
Step 1: GRPO outcome advantage (seq-level)
  A_seq[i] = R_i  -  mean_{j ∈ group(i)} R_j

Step 2: per-sequence normalize teacher surprise
  g_t      = KL( P_T(·|x, y_≤t) ‖ P_T(·|x, y_<t) )    # teacher forward-Bayes surprise
  ĝ_t      = g_t / mean_t(g_t)                          # per-sequence normalize on response_mask
  ĝ_t      = clip(ĝ_t, max=max_ratio)                   # 上限保护

Step 3: v2 新增 - clip 后重归一化（可选）
  ĝ_t      = ĝ_t / mean_t(ĝ_t)                         # 保 mean(ĝ)=1（renormalize_after_clip=True）

Step 4: 合成 token-level advantage + length penalty
  A_t      = A_seq · ĝ_t · length_scale(L)
  length_scale(L) = min(L / min_response_length, 1.0)    # 短序列线性衰减（默认 50 token）
```

详见 [prior_shift.py](file:///Users/awesome_jimmy/lazada/SDPO/verl/trainer/ppo/bayesian_credit/prior_shift.py#L13-L16)。

- **诊断结论**：
  1. **Length collapse（致命）**：response 从 372 token 坍缩到 18 token，原因链：
     - max_ratio=10 → 极端 surprise token advantage 放大 10x
     - 短序列 token reward density 高（同一 R 摊到 18 vs 372 token）
     - 正反馈循环：缩短 → reward/token ↑ → 梯度 ↑ → 进一步缩短
  2. **Advantage 量级异常**：critic/advantages range [-3.07, 2.65] vs RLSD [-1.05, 1.05]
  3. **Val acc 过峰塌方**：best 在 step 130，剩余 120 步全是负贡献（浪费 48% 算力）
  4. **critic/rewards/mean 上升**：0.001 → 0.026（+27x），但不是好事，是 length collapse 的副作用

- **根因追溯**：
  ```
  max_ratio=10（过大）
  → ĝ_t clip 后 mean(ĝ) ≠ 1（归一化被破坏）
  → 梯度量级不恒定
  → 短序列每 token 优势密度高
  → 模型学会缩短回答
  → 正反馈循环
  ```

- **修复方案**（v2）：
  - P0：max_ratio 10→3（截断极端 weight）
  - P0：min_response_length=50 + linear penalty（抵抗 length collapse）
  - P1：renormalize_after_clip=True（clip 后保 mean(ĝ)=1）

## 3.1 TGDI v3 实验设计（2026-05-16 新增）

### 论文叙事（Bayesian Credit Assignment Tier 3）

```
Tier 1: Prior-Shift (相关性归因)        ← v1 失败 / v2 在跑
Tier 2: Posterior-Shift (反向归因)      ← 暂缓
Tier 3: Causal Intervention (因果归因)  ← 本次提交首发
```

核心论点：**student 的错误 = 某个 token 偏离了 teacher 的认知**。因果验证：让 teacher 在那个点接管，看 reward 是否回升。ΔR 是 RLSD 论文 Theorem 1 中允许的 "magnitude signal"（dense + reliable），不走 OPSD 的 leakage 路径。

### Phase 1 vs Phase 2

| 项 | Phase 1（已实现）| Phase 2（待实现）|
|---|---|---|
| `enable_intervention` | False | True |
| 实际 rollout | 单轮 student rollout | student + teacher 接管 + tail |
| ΔR 通道 | 始终 0 | R(y') - R(y_s) |
| 等价于 | prior_shift v2 | 真实 v3-full |
| 新工程量 | 0（已交付） | dp_actor +30 行 + intervention_rollout._do_real_intervention +250 行 |
| 用途 | 验证 plumbing + baseline 11 个 intervention/* 指标 | 真实因果训练，论文主结果 |

### t\* 选择策略消融（3 个 Phase 1 jobs）

## 4. 主假说与理论分析

`self_teacher_advantage` 用 teacher prob 构造 advantage：

- teacher 高概率 token → A > 0
- teacher 低概率 token → A < 0

但 EMA r=0.1 + rel_L2≈1e-4 + cos_sim=1.0 → **teacher 几乎就是 student**：

> teacher 高 prob ⇔ student 高 prob，teacher 低 prob ⇔ student 低 prob

代入 PG 梯度 `A · ∇log π(a)`：

```
高 prob token (A > 0) → 推得更高
低 prob token (A < 0) → 推得更低
```

这就是 **V_CE 的本质**（点级模式追逐），与 SDPO 的分布级 JSD 截然不同 —— SDPO 之所以不崩，是因为它对整个分布形状有约束。

**放大器**：
- group norm + clip=3 → advantage 全程贴边 → 主假说方向被无差别放大
- `use_vce=True` 显式叠加 V_CE 字面项
- entropy_coeff=0.001 太弱
- 负 advantage 稀缺（teacher 几乎处处 ≥ student），剩下主要是正向 mode-seeking

### Tier 1（**新主叙事首发：Prior-Shift POC**）⭐

**核心 idea**：用 teacher 的 forward Bayesian surprise 作为 token-level credit，把 outcome reward 按 surprise 分配。

**公式**（v2）：
```
A_seq[i] = R_i  -  mean_{j ∈ group(i)} R_j
g_t      = KL( P_T(·|x, y_≤t) ‖ P_T(·|x, y_<t) )       # token-level surprise
ĝ_t      = g_t / mean_t(g_t)                             # group-level normalize
ĝ_t      = clip(ĝ_t, max=max_ratio)                      # 上限保护
ĝ_t      = ĝ_t / mean_t(ĝ_t)                             # v2 新增: renormalize
A_t      = A_seq · ĝ_t · length_scale(L)                 # length penalty
```

**vs RLSD 的差异**：
- RLSD：`A_t = A · (P_T/P_S)^sign(A)` —— **基于 teacher-student diff**
- Ours：`A_t = A_seq · ĝ_t · length_scale` —— **基于 teacher 自己的 belief update**，与 student 无关
- 解释：RLSD 是 "模仿 teacher"，我们是 "找 teacher 认为信息含量大的位置"

**工程实现要点**：
- 仅需 +1 次 teacher forward（拿 logits over full vocab）
- 复用现有 `entropy_gate` 基础设施作 mask 框架
- 入口：`nebula_scripts/prior_shift/prior_shift_sciknoweval_parametric.sh`
- adv_mode 新增 `prior_shift`，与 `self_teacher` 并列

**v1 失败根因**：
- max_ratio=10 过大 → advantage 量级 3x RLSD → 梯度 10x → length collapse
- clip 后归一化被破坏 → 梯度量级不恒定

**v2 修复**：
- max_ratio 10→3
- renormalize_after_clip=True（保 mean(ĝ)=1）
- min_response_length=50 + linear penalty

---

### Tier 2（**新主叙事扩展：Posterior-Shift**）

**核心 idea**：teacher 看到 outcome 后反推每个 token 的重要性（hindsight credit）。

**公式**：
```
h_t = log P_T(y_t | x, y, r=R) − log P_T(y_t | x, y_<t)
```

**实现路径**：
- 把 `<reward=R>` 作为 prefix token 喂给 teacher（或在 system prompt 加入 outcome 描述）
- 需要先在 teacher 训练 / inference pipeline 加 reward-conditioned forward 接口
- 与 Tier 1 比，多 1 次 teacher forward

**实验顺序**：
- C1：Posterior-Shift 单独跑
- C2：Prior + Posterior 加权融合（α·g_t + (1−α)·h_t）
- C3：与 RLSD baseline 对比 val_acc / entropy / response_length

## 6. Related Work：RLSD 撞车与 reframe

- **论文**：Self-Distilled RLVR (RLSD)，arXiv:2604.03128v2
- **核心公式**：`Â_t = A · clip((P_T/P_S)^sign(A), 1−ε_w, 1+ε_w)`
- **理论贡献**：Bayesian evidence ratio (Theorem 4)、impossibility trilemma (Theorem 3)、leakage-free 证明 (Theorem 5)、OPSD mutual information gap (Theorem 1)
- **实验**：Qwen3-VL-8B-Instruct + MMFineReason-123K，5 个多模态 benchmark，比 GRPO +2.32%
- **撞车结论**：早期"prob diff + sign(R)"思路与 RLSD 同构（线性近似）；前向 prob ratio 这条路已被堵死

**Reframe 后的差异化定位**：

| 维度 | RLSD | Ours (Prior+Posterior Shift) |
| --- | --- | --- |
| credit 来源 | teacher-student log-ratio | teacher 自己的 belief shift |
| 因果方向 | 前向（causal） | 前向 + 反向（hindsight） |
| 是否需要 student 信号 | 是 | 否（解耦 teacher）|
| Bayesian 视角 | prior 模仿 | prior + posterior 双视角 |
| benchmark | 多模态 Qwen3-VL | 文本任务（sciknoweval / math）|
| teacher 形式 | hard sync 每 10 步 | EMA + 实测对比 |
